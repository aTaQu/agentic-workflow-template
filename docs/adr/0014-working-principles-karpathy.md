# Behavioral principles layer, adapted from Karpathy's guidelines

`AGENTS.md` carries a `## Working principles` section — Think Before Coding, Simplicity First,
Surgical Changes, Goal-Driven Execution, and Verify-before-handing-over — the always-on behavioral
discipline that complements the workflow loop. Adapted (reworded, stack-neutral) from Andrej
Karpathy's coding guidelines (github.com/multica-ai/andrej-karpathy-skills) and credited in-line.
Considered vendoring the upstream `karpathy-guidelines` skill instead (modular, but model-invoked
on demand is the wrong shape for always-on discipline — it would slip). Reworded rather than copied
verbatim because the upstream repo carries no license; #5 is carried over from the sodyba exemplar's
own addition, genericized.
