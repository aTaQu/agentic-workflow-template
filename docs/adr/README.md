# Architecture Decision Records

One file per decision: `NNNN-slug.md`, sequentially numbered. Keep them short — a title plus 1–3
sentences (what's the context, what did we decide, and why). Format:
[`ADR-FORMAT.md`](../../.claude/skills/grill-with-docs/ADR-FORMAT.md).

Record a decision as an ADR only when **all three** hold: it's **hard to reverse**, **surprising
without context**, and **the result of a real trade-off**. Otherwise skip it — you'll just reverse
an easy decision, nobody wonders about an unsurprising one, and there's nothing to record when there
was no alternative.

> ADRs 0001–0013 here are **this template's own** design decisions — they explain why the workflow
> is shaped the way it is. When you start a real project from the template, read them once, then
> clear them (or keep them as a reference) and start your own numbering at 0001.
